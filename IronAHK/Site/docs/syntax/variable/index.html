Variables

A variable is an identifier which refers to data that can be modified.

Form

A valid identifier can be any alphanumeric string between 1 and 64 characters in length. It must not be one of the reserved keywords or contain symbols.

Types

Every variable has an underlying type which describes the nature of the data it stores. The following table lists the primitive types commonly used:

Type CLR Type Description Example
String System.String Set of Unicode characters. txt := "Hello World"
Numeric System.Double Floating point number. pi := 3.14
Boolean System.Boolean True or false. error := false
Array System.Object[] List of values. primes := [ 1, 3, 5, 7, 11 ]
Object System.Dictionary Associative array. result := { success : true, code : 200 }
Function System.Delegate Function. f := { append : myFunction }, result := f.append("test")

In certain cases a variable may be any other CLR type. However unlike the CLR, all value types are automatically boxed.

Arrays

An array stores multiple values (called elements) which can be of different types all within a single variable. Each element has a numeric index starting from zero.
Square brackets [ ] are used to create arrays, refer to specific elements and extend arrays:

primes := [ 1, 3, 5, 7 ] ; create an array with a few initial values
MsgBox("Third prime number: " primes[2]) ; the second index is the third element, since indexes start at zero

primes[] := 11 ; extend the array by one element and store this new value

Objects

An object is similar to an array but elements have a key name rather than a numeric index. They can also be called associative arrays, or variables within variables.

The syntax is identical to JSON:

book := { author : "Charles Darwin", price : 11.90 } ; create an object
MsgBox("Author: " book.author) ; reference an element
book["price"] += 1.10 ; same as book.price
book.title := "The Origin of Species" ; create a new element named title

Scope

With the exception of local variables in functions, all variables are global - their contents may be read or altered by any part of the script.

Variables do not need to be declared, they are automatically created whenever they are referenced.

To delete a variable or array element set its value to null.
